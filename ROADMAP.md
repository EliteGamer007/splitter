# Project Roadmap

This document outlines the planned trajectory for the Splitter project. The timeline is subject to change based on community feedback and developmental milestones.

## ✅ Phase 1: Foundations (Complete)
- [x] Functional Go/Echo Backend.
- [x] Basic Next.js Frontend.
- [x] Password-based Authentication.
- [x] Initial Decentralized Identity (DID) support.
- [x] Database schema for Posts, Users, and Likes.

## 🚧 Phase 2: Social Core & Scaling (Current)
- [x] **Full-Text Hashtag Discovery**: Real-time trending and tag-based search.
- [x] **Threaded Recursive Replies**: Deep nested conversations with efficient frontend rendering.
- [x] **Federation MVP**: Basic ActivityPub Actor and WebFinger implementation.
- [x] **Stories**: 24-hour media sharing with view tracking.
- [x] **Comprehensive Documentation**: OpenAPI expansion and specialized technical guides.
- [ ] **E2EE Messaging Improvements**: Robust multi-device key synchronization.
- [ ] **Performance Optimization**: Advanced caching Layer for high-traffic instances.

## 🚀 Phase 3: Advanced Federation & Privacy (Upcoming)
- [ ] **Cross-Instance Search**: Discovering users and content across the wider Fediverse.
- [ ] **Defederation Tools**: More granular instance-level moderation and reputation logic.
- [ ] **Encrypted Groups**: Private, multi-user encrypted messaging using double-ratchet logic.
- [ ] **Media Optimization**: Auto-resizing and CDN integration for large media attachments.
- [ ] **Community Plugins**: Support for custom backend hooks and frontend widgets.

## 🔭 Phase 4: Long-Term Vision
- [ ] **Native Mobile Apps**: Dedicated iOS and Android clients.
- [ ] **Decentralized Storage Integration**: Optional support for IPFS or Arweave for permanent media hosting.
- [ ] **Voice & Video Support**: Peer-to-peer federated real-time communication.
- [ ] **Monetization Tools**: Optional tip-jars and subscription models for creators.
